# Git Worktree 一覧

## 概要

worktree の一覧と状態を取得し、報告用テーブルで出力する。
**GitKraken MCP を最優先**で使い、MCP 未対応の操作のみ `git` をフォールバックで使う。

## 手順

### 1. worktree 一覧の取得（GitKraken MCP 優先）

- **まず** GitKraken MCP の `git_worktree`（`action: "list"`）を、メインリポジトリの `directory` を指定して実行する。
- MCP が使えない・結果が不足している場合のみ、フォールバックで次を実行する。

```sh
git worktree list
```

- active な作業ディレクトリを誤って消さないよう、一覧で確認する。

### 2. 各 worktree の状態収集（GitKraken MCP 優先）

各 worktree ごとに次を確認する。**GitKraken MCP を優先**して使う。

- **未コミット変更**: GitKraken MCP の `git_status` を、対象 worktree のディレクトリ（`directory`）で実行し、変更の有無・件数を把握する。
- **lock 状態**: 一覧に `locked` と出ているかで判定する。
- **ディレクトリ存在**: 当該パスが実在するか（削除済みで参照だけ残っている場合は「なし」とする）。

MCP が使えない場合は、該当ディレクトリで `git status` や `ls` でフォールバックする。

### 3. 報告フォーマット

次のテーブル形式で一覧を報告する。

```
┌───────────────────────────────────────────────────────┬────────────────────────────────────────────────┬────────────────┬──────┬──────────────┬────────────────┬────────────────────────────────────┐
│ worktree ディレクトリ                                 │ ブランチ                                       │ 未コミット変更 │ lock │ ディレクトリ │ 削除可否       │ 備考                               │
├───────────────────────────────────────────────────────┼────────────────────────────────────────────────┼────────────────┼──────┼──────────────┼────────────────┼────────────────────────────────────┤
│ <ディレクトリ名>                                      │ <ブランチ名 or (detached HEAD)>                │ <件数 or ->    │ -/🔒 │ 存在/なし    │ ✅ 削除可 / ⚠️ prune のみ / ❌ 要対応 │ クリーン / 未コミットあり / など   │
└───────────────────────────────────────────────────────┴────────────────────────────────────────────────┴──────────────┴──────┴──────────────┴────────────────┴────────────────────────────────────┘
```

- **削除可否**の目安:
  - **✅ 削除可**: 未コミット変更なし・lock なし・ディレクトリ存在
  - **⚠️ prune のみ**: ディレクトリがすでになく、参照だけ残っている場合（`worktree prune` のみ提案）
  - **❌ 要対応**: 未コミット変更あり、または lock あり（先にコミット/stash または unlock が必要）
